第9章 パフォーマンス作法

Minecraft Wikiの「Tutorial: Optimizing a data pack」とコミュニティ実践からの要点。

9.1 セレクタ

  • @e は最小限。型を必ず指定(@e[type=marker])し、可能なら tag= で第一絞り込み。 セレクタ引数は概ね「安い順に絞れる」よう type → tag → distance/scores の意識で。
  • 同じセレクタを複数行で繰り返さない → asで関数に入って@s化(3.2)。
  • 遠くで動く必要のない処理には distance=..N を付ける。大量エンティティ相手なら 「プレイヤー周辺に定期的にactiveタグを付け、以後はtag=activeで選ぶ」方式が distance連打より安い(Wiki推奨テク)。
  • @a は @e[type=player] より速い(プレイヤーリストだけ走査する)。

9.2 NBTを避ける階段

  1. セレクタnbt=引数 → predicateへ(equipment, vehicle等は専用フィールドがある)
  2. アイテム判定 → execute if items entity @s weapon.mainhand <item>(1.20.5+、最速級)
  3. どうしてもNBT加工 → storageへコピーして加工(プレイヤーNBT直接操作は最悪コスト)
  4. アイテム改変 → data modifyでなく item_modifier / item replace

なお「predicateの中でnbtキーだけ使う」のは速くならない(Ellivers)。 predicateが速いのは構造化フィールドを使った時だけ。

9.3 tickを軽くする

  • tick.jsonには分配関数1つ(3.1)。分配関数の先頭で「そもそも今やることがあるか」を スコア1個で判定して早期return。
  • 毎tick必要ない処理は schedule 自己ループで 2t/5t/20t に間引く。 エンティティ個別なら predicate の periodic_tick も使える(セレクタは走るのでscheduleより劣る)。
  • 重い計算は複数tickに分割する(時分割)。tick予算(50ms)を意識する。

9.4 計測してから最適化する

  • /debug start → /debug stop のプロファイル結果で関数ごとの時間を見る (commandFunctions/scheduledFunctions の項目)。
  • F3のTPSグラフ//perf、または計測用データパック(Kragast's Benchmark等)でベンチ。
  • 推測で「速い書き方」に書き換えて可読性を落とすのは本末転倒。遅いと分かった場所だけ直す。 この点も『リーダブルコード』と同じ。

第10章 互換性・複数データパック共存

サーバー(にわとりMCのようにロビー+PVP等)で複数パックを同居させるなら必須の章。

10.1 絶対ルール

  • minecraft名前空間のtick/loadタグに "replace": true を書かない(2.2)
  • 全ての名前(objective/tag/storage/team/bossbar/fakeplayer)に名前空間プレフィックス
  • バニラ挙動の上書き(loot table等)は「合理的な範囲」に留め、README/descriptionに明記する

10.2 共通タグ規約に乗る

  • smithed.entity: データパックが生成したエンティティ(「バニラのゾンビ」等を対象とする 他パックの処理から除外してもらう)
  • smithed.strict: 他パックは一切触らないでほしいエンティティ
  • Bookshelfの bs.persistent のように「killしないで」系タグを尊重する (掃除コマンドは kill @e[type=item] でなく kill @e[type=item,tag=!nw.keep,tag=!smithed.strict] 的に)

自作エンティティには自分のタグ+smithed.entityを両方付けるのが行儀の良い作法。

10.3 カスタムアイテムの識別

custom_data に自名前空間で {niwatori:{id:"rating_sword"}} 形式のIDを持たせ、 判定は必ずこのIDで行う。Item名やCustomModelDataでの判定は、リソパ更新や 他パックとの併用で壊れる。(旧CMD時代はSmithedのCMDプレフィックス規約があったが、 現行はcustom_data/item_modelコンポーネントで名前空間分離するのが素直。)
