第14章 テストとデバッグ

14.1 debugネームスペース/フォルダを常設する

function/debug/
├── dump_player.mcfunction    # @sの主要スコアとstorageをtellraw
├── give_test_kit.mcfunction
└── reset_all.mcfunction
  • デバッグ出力は @a[tag=nw.debug] にだけ流す(本番プレイヤーに漏らさない)。
  • Bookshelf Devのように「ログ・変数ダンプ専用の開発用パック」を分離する構成も良い。

14.2 表明(アサーション)を書く

「起きないはず」の状態を検知したら管理者へ通報する行を要所に入れる:

execute if score #arena.state nw.var matches 4.. run \
    tellraw @a[tag=nw.admin] {"text":"[BUG] arena.stateが範囲外","color":"red"}

14.3 テスト関数

純粋な計算関数(レーティング計算等)は test/ に期待値比較の関数を書き、 手動 or CI(ヘッドレスサーバー起動)で叩ける形にしておく。 execute store + if score ... matches 期待値 + 結果tellraw、で最低限のユニットテストになる。

第15章 アンチパターン集

アンチパターンなぜ悪い代わりに
tick.jsonに関数を直接何個もぶら下げる実行順が読めない、全体像が消える分配関数1つ(3.1)
replace: true をtick/load.jsonに他パック全滅絶対禁止(2.2)
プレフィックス無しobjective/tag衝突名前空間プレフィックス(第1章)
1関数に複数の実行文脈~ ~ ~が誰か分からない@s中心+分割(第5章)
巨大マクロ関数全行が毎回再解析マクロ行だけ隔離(6.3)
ユーザー入力→マクロ直結任意コマンド実行ID化・検証(6.5)
nbt={SelectedItem:...}セレクタ激重predicate / if items(9.2)
条件なし自己再帰チェーン上限まで暴走ガード節必須(8.3)
scheduleにreplace無しreload毎にループ増殖replace指定(11.2)
kill @e[type=marker]等の雑掃除他パック・自パックの基盤を破壊タグで限定(10.2)
0初期化を前提にしたmatches ..0未定義スコアに効かない明示的初期化(7.4)
コメントとコードの乖離嘘ドキュメント変更時にヘッダも更新(4.6)
デバッグsayの放置本番でチャット汚染debugタグ宛てtellraw(14.1)
生成ファイルの手編集次の生成で消える生成物隔離(3.5)
