リーダブル・データパック

〜 良いデータパックの書き方 完全ガイド 〜

対象バージョン: Java Edition 1.21.x(1.20.2以降のマクロ対応を前提。26.x系への移行時の注意も併記)

このドキュメントは『リーダブルコード』のようなコーディング作法本の「Minecraftデータパック版」として、 コミュニティの規約(Smithed Conventions / MC Datapacks Conventions / Gamemode 4 標準 / Bookshelf コントリビューションガイド / IMP-Doc / Minecraft Wiki 最適化チュートリアル 等)と、 GitHub上の著名データパック(Gamemode 4, Bookshelf, PlayerDB, Vanilla Tweaks系 等)の実践から 「こう書かれたデータパックは読みやすく、壊れにくく、速い」というルールをまとめたものです。

一番大事な原則(リーダブルコードと同じ)

コードは「書く時間」より「読む時間」の方が圧倒的に長い。 mcfunctionは特に 実行文脈(誰が・どこで実行しているか)がファイルに書かれていない 言語なので、 「未来の自分が3秒で文脈を復元できるか」を常に基準にする。

目次

第1章

命名規則

lower_snake_case、動詞始まりの関数名、objective・タグ・storageなど全IDの命名ルール。

 第2〜3章

名前空間とディレクトリ構造

名前空間は署名。minecraft名前空間の扱い、レイヤー分割、1ファイル1責務の考え方。

 第4〜5章

コメントと実行文脈

全関数のヘッダコメント、IMP-Doc、「@s中心主義」による実行文脈の設計。

 第6章 ★

マクロ関数の設計

本ガイドの目玉。マクロ実体の命名・引数規約・隔離・セキュリティ・使いどころの選択表。

 第7〜8章

変数管理と制御構造

scoreboard / storage の使い分け、定数プール、早期return、ループ、イベント駆動。

 第9〜10章

パフォーマンスと互換性

セレクタ最適化、NBTを避ける階段、計測の作法、複数データパック共存のルール。

 第11〜13章

運用・配布・ツール

load / tick / install / uninstall の標準セット、pack.mcmeta、Git管理、開発環境とCI。

 第14〜15章

テストとアンチパターン

debugフォルダの常設、アサーション、テスト関数、やってはいけないパターン14選。

 第16〜17章

チェックリストと参考文献

提出前のセルフチェック項目と、このガイドの元になった一次ソース一覧。

このガイドの読み方

各章は独立して読めますが、初めての方は第1章から順に読むのがおすすめです。 すでにデータパックを書いている方は、アンチパターン集と チェックリストから入って、引っかかった項目の章に戻ると効率的です。

このガイドは1.21.x時点のコミュニティ標準をまとめたものです。バージョン更新(特に26.x系の pack_format刷新・NBT型厳格化)で細部は変わるため、移行時は各一次ソースを再確認してください。
